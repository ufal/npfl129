- The computation is performed analogously:
  $$\begin{aligned}
    h_i &= f\left(‚àë\nolimits_j x_j w^{(h)}_{j,i} + b^{(h)}_i\right), \\
    y_i &= a\left(‚àë\nolimits_j h_j w^{(y)}_{j,i} + b^{(y)}_i\right),
  \end{aligned}$$
  or in matrix form
  $$\begin{aligned}
    ‚Üíh &= f\Big(‚Üíx^T ‚áâW^{(h)} + ‚Üíb^{(h)}\Big), \\
    ‚Üíy &= a\Big(‚Üíh^T ‚áâW^{(y)} + ‚Üíb^{(y)}\Big),
  \end{aligned}$$
  and for a batch of inputs $‚áâH = f\Big(‚áâX ‚áâW^{(h)} + ‚Üíb^{(h)}\Big)$ and $‚áâY = a\Big(‚áâH ‚áâW^{(y)} + ‚Üíb^{(y)}\Big)$.

# Training MLP ‚Äì Computing the Derivatives

![w=20%,f=right](../04/mlp.svgz)

Assume we have an MLP with input of size $D$, weights $‚áâW^{(h)} ‚àà ‚Ñù^{D √ó H}$,
$‚Üíb^{(h)} ‚àà ‚Ñù^H$, hidden layer of size $H$ and activation $f$ with weights
$‚áâW^{(y)} ‚àà ‚Ñù^{H √ó K}$, $‚Üíb^{(y)} ‚àà ‚Ñù^K$, and finally an output layer of size $K$ with
activation $a$.

~~~
To compute the gradient of the loss $L$ with respect to all weights, proceed gradually:
- first compute $\frac{‚àÇL}{‚àÇ‚Üíy}$,

~~~
- then compute $\frac{‚àÇ‚Üíy}{‚àÇ‚Üíy^\mathit{(in)}}$, where $‚Üíy^\mathit{(in)}$ are the
  inputs to the output layer (i.e., before applying activation function $a$;
  in other words, $‚Üíy = a(‚Üíy^\mathit{(in)})$),
~~~
- then compute $\frac{‚àÇ‚Üíy^\mathit{(in)}}{‚àÇ‚áâW^{(y)}}$ and $\frac{‚àÇ‚Üíy^\mathit{(in)}}{‚àÇ‚Üíb^{(y)}}$,
  which allows us to obtain $\frac{‚àÇL}{‚àÇ‚áâW^{(y)}} = \frac{‚àÇL}{‚àÇ‚Üíy}
  ‚ãÖ \frac{‚àÇ‚Üíy}{‚àÇ‚Üíy^\mathit{(in)}} ‚ãÖ \frac{‚àÇ‚Üíy^\mathit{(in)}}{‚àÇ‚áâW^{(y)}}$ and analogously
  $\frac{‚àÇL}{‚àÇ‚Üíb^{(y)}}$,
~~~
- followed by $\frac{‚àÇ‚Üíy^\mathit{(in)}}{‚àÇ‚Üíh}$ and $\frac{‚àÇ‚Üíh}{‚àÇ‚Üíh^\mathit{(in)}}$,
~~~
- and finally using $\frac{‚àÇ‚Üíh^\mathit{(in)}}{‚àÇ‚áâW^{(h)}}$ and
  $\frac{‚àÇ‚Üíh^\mathit{(in)}}{‚àÇ‚Üíb^{(h)}}$ compute $\frac{‚àÇL}{‚àÇ‚áâW^{(h)}}$ and
  $\frac{‚àÇL}{‚àÇ‚Üíb^{(h)}}$.

# Hidden Layer Interpretation and Initialization

![w=90%,h=center,mw=45%,f=right](lr_linearity.svgz)

One way how to interpret the hidden layer is:
- the part from the hidden layer to the output layer is the previously
  used generalized linear model (linear regression, logistic regression, ‚Ä¶);

~~~
- the part from the inputs to the hidden layer can be considered as automatically
  constructed features. The features are a linear mapping of the input values
  followed by a nonlinearity, and the theorem on an upcoming slide proves they can
  always be constructed to achieve as good a fit of the training data as is required.

~~~
Note that the weights in an MLP must be initialized randomly. If we used just
zeros, all the constructed features (hidden layer nodes) would behave
identically and we would never distinguish them.

Using random weights corresponds to starting with random features, which allows the SGD
to make progress (improve the individual features).

# Universal Approximation Theorem '89

Let $œÜ(x):‚Ñù ‚Üí ‚Ñù$ be a nonconstant, bounded and nondecreasing continuous function. \
(Later a proof was given also for $œÜ = \ReLU$ and even for any nonpolynomial
function.)

~~~
For any $Œµ > 0$ and any continuous function $f: [0, 1]^D ‚Üí ‚Ñù$, there exists
$H ‚àà ‚Ñï$, $‚Üív ‚àà ‚Ñù^H$, $‚Üíb ‚àà ‚Ñù^H$ and $‚áâW ‚àà ‚Ñù^{D√óH}$ such that if we denote
$$F(‚Üíx) = ‚Üív^T œÜ(‚Üíx^T ‚áâW + ‚Üíb) = ‚àë_{i=1}^H v_i œÜ(‚Üíx^T ‚Üí{W_{*, i}} + b_i),$$
where $œÜ$ is applied elementwise,
~~~
then for all $‚Üíx ‚àà [0, 1]^D$:
$$|F(‚Üíx) - f(‚Üíx)| < Œµ.$$

# Universal Approximation Theorem for ReLUs

Sketch of the proof:

~~~
- If a function is continuous on a closed interval, it can be approximated by
  a sequence of lines to arbitrary precision.

![w=50%,h=center](universal_approximation_example.png)

~~~
- However, we can create a sequence of $k$ linear segments as a sum of $k$ ReLU
  units ‚Äì on every endpoint, a new ReLU starts (i.e., the input ReLU value is
  zero at the endpoint) with a tangent which is the difference between the
  target tangent and the tangent of the approximation until this point.

# Constrained Optimization

Given a function $f(‚Üíx)$, we can find a minimum/maximum with respect to a vector
$‚Üíx ‚àà ‚Ñù^D$ by investigating the critical points $‚àá_{‚Üíx} f(‚Üíx) = 0$.

~~~
Consider now finding a minimum subject to a constraint $g(‚Üíx) = 0$.

~~~
![w=40%,f=left](lagrange_example.png)

\
\
\
On the left, there is an example with $f(x, y) = x + y$ and
the constraint $x^2 + y^2 = 1$, which can be represented
as $g(x, y) = x^2 + y^2 - 1$.

---
# Lagrange Multipliers ‚Äì Equality Constraints

<div style="float: right; width: 26%">![w=95%,h=center](lagrange_example.png)<br>![w=100%](lagrange_equalities.svgz)</div>

Let $f(‚Üíx): ‚Ñù^D ‚Üí ‚Ñù$ be a function. We seek its minimum subject
to an equality constraint $g(‚Üíx) = 0$ for $g(‚Üíx): ‚Ñù^D ‚Üí ‚Ñù$.

~~~
- Note that $‚àá_{‚Üíx} g(‚Üíx)$ is orthogonal to the surface of the constraint, because
  if $‚Üíx$ and a nearby point $‚Üíx+‚ÜíŒµ$ lie on the surface, from the Taylor
  expansion $g(‚Üíx+‚ÜíŒµ) ‚âà g(‚Üíx) + ‚ÜíŒµ^T ‚àá_{‚Üíx} g(‚Üíx)$ we get $‚ÜíŒµ^T ‚àá_{‚Üíx} g(‚Üíx) ‚âà 0$.

~~~
- For the desired minimum, $‚àá_{‚Üíx} f(‚Üíx)$ must also be orthogonal to the constraint
  surface (or else moving in the direction opposite to the derivative would decrease the
  value).

~~~
- Therefore, there must exist $Œª$ such that $‚àá_{‚Üíx} f = Œª‚àá_{‚Üíx} g$.

~~~
Consequently, the sought minimum either fulfills $‚àá_{‚Üíx} f - Œª‚àá_{‚Üíx} g = 0$ for some $Œª$,
or it is an unconstrained minimum ‚Äì in that case, the equation also holds with
$Œª=0$.

---
# Minimization ‚Äì Equality Constraint

Let $f(‚Üíx): ‚Ñù^D ‚Üí ‚Ñù$ be a function that has a minimum (or a maximum) in $‚Üíx$
subject to the equality constraint $g(‚Üíx) = 0$. Assume that both $f$ and $g$ have
continuous partial derivatives and that $‚àá_{‚Üíx} g(‚Üíx) ‚â† 0$.

~~~
Then there exists a $Œª ‚àà ‚Ñù$, such that the **Lagrangian function**
$$ùìõ(‚Üíx, Œª) ‚âù f(‚Üíx) - Œªg(‚Üíx)$$
has a zero gradient in both $‚Üíx$ and $Œª$.

~~~
In detail,
- $\frac{‚àÇùìõ}{‚àÇŒª} = 0$ leads to $g(‚Üíx)=0$;
~~~
- $‚àá_{‚Üíx} ùìõ = 0$ is the previously derived $‚àá_{‚Üíx} f - Œª‚àá_{‚Üíx} g = 0$.

---
# Minimization ‚Äì Multiple Equality Constraints

_We can use induction if there are multiple equality constraints, resulting in
the following generalization._

Let $f(‚Üíx): ‚Ñù^D ‚Üí ‚Ñù$ be a function that has a minimum (or a maximum) in $‚Üíx$
subject to equality constraints $g_1(‚Üíx) = 0, ‚Ä¶, g_m(‚Üíx) = 0$. Assume that
$f, g_1, ‚Ä¶, g_m$ have continuous partial derivatives and that the gradients
$‚àá_{‚Üíx} g_1(‚Üíx), ‚Ä¶, ‚àá_{‚Üíx} g_m(‚Üíx)$ are linearly independent.

~~~
Then there exist $Œª_1 ‚àà ‚Ñù, ‚Ä¶, Œª_m ‚àà ‚Ñù$, such that the **Lagrangian function**
$$ùìõ(‚Üíx, ‚ÜíŒª) ‚âù f(‚Üíx) - ‚àë_{i=1}^m Œª_i g_i(‚Üíx)$$
has a zero gradient in both $‚Üíx$ and $‚ÜíŒª$.

~~~
This strategy of finding constrained minima is known as the **method of Lagrange
multipliers**.

---
# Example of Minimization with Equality Constraint

Assume we want to find a categorical distribution $‚Üíp = (p_1, ‚Ä¶, p_n)$ with maximum
entropy.

~~~
Then we want to minimize $\textcolor{red}{-H(‚Üíp)}$ under the constraints
- $p_i ‚â• 0$ for all $i$,
~~~
- $\textcolor{darkgreen}{‚àë_{i=1}^n p_i = 1}$.

~~~
Ignoring the first constraint for the time being, we form the Lagrangian
$$ùìõ = \textcolor{red}{\bigg(‚àë_i p_i \log p_i\bigg)} - \textcolor{darkgreen}{Œª\bigg(‚àë_i p_i - 1\bigg)}.$$

~~~
Computing the derivative with respect to $p_i$ and setting it equal to zero, we get
$$0 = \frac{‚àÇùìõ}{‚àÇp_i} = \textcolor{red}{1‚ãÖ\log(p_i) + p_i ‚ãÖ \frac{1}{p_i}} - \textcolor{darkgreen}{Œª}
    = \textcolor{red}{\log(p_i) + 1} - \textcolor{darkgreen}{Œª}.$$

~~~
Therefore, all $p_i = e^{Œª-1}$ must be the same, and the constraint $‚àë_{i=1}^n p_i = 1$
yields $p_i = \frac{1}{n}$.

# Derivation of Softmax using Maximum Entropy

Let $ùïè = \{(‚Üíx_1, t_1), (‚Üíx_2, t_2), ‚Ä¶, (‚Üíx_N, t_N)\}$ be training data
of a $K$-class classification, with $‚Üíx_i ‚àà ‚Ñù^D$ and $t_i ‚àà \{1, 2, ‚Ä¶, K\}$.

~~~
We want to model it using a function $œÄ: ‚Ñù^D ‚Üí ‚Ñù^K$
so that $œÄ(‚Üíx)$ gives a distribution of classes for input $‚Üíx$.

~~~
We impose the following conditions on $œÄ$:
- $\displaystyle \textrm{for~}1 ‚â§ k ‚â§ K\!:~ œÄ(‚Üíx)_k ‚â• 0,$

~~~
- $\displaystyle \phantom{\textrm{for~}1 ‚â§ k ‚â§ K\!:~} ‚àë_{k=1}^K œÄ(‚Üíx)_k = 1,$
~~~
- $\displaystyle \textrm{for~}1 ‚â§ k ‚â§ K\!:~ ‚àë_{i=1}^{\smash N} œÄ(‚Üíx_i)_k ‚Üíx_i = ‚àë_{i=1}^{\smash N} \Big[t_i = k\Big] ‚Üíx_i.$

---
# Derivation of Softmax using Maximum Entropy

There are many such $œÄ$, one particularly bad is
$$œÄ(‚Üíx) = \begin{cases}
  ‚Üí1_{t_i}&\textrm{if there exists }i: ‚Üíx_i = ‚Üíx, \\
  ‚Üí1_1&\textrm{otherwise},\end{cases}$$
where $‚Üí1_i$ is a one-hot encoding of $i$ (vector of zeros, except for position $i$, which is equal to 1).

~~~
Therefore, we want to find a more **general** $œÄ$ ‚Äì consequently, we turn to the principle of maximum
entropy and search for $œÄ$ with maximum entropy.

---
# Derivation of Softmax using Maximum Entropy

We want to minimize $\textcolor{red}{-‚àë_{i=1}^N H(œÄ(‚Üíx_i))}$
given
- $\textrm{for~}1 ‚â§ i ‚â§ N,~ 1 ‚â§ k ‚â§ K\!:~ œÄ(‚Üíx_i)_k ‚â• 0$,
- $\textrm{for~}1 ‚â§ i ‚â§ N: \hphantom{,~ 1 ‚â§ k ‚â§ K\!}~ \textcolor{darkgreen}{‚àë_{k=1}^K œÄ(‚Üíx_i)_k = 1}$,
- $\textrm{for~}1 ‚â§ j ‚â§ D,~ 1 ‚â§ k ‚â§ K\!:~ \textcolor{magenta}{‚àë_{i=1}^N œÄ(‚Üíx_i)_k x_{i,j} = ‚àë_{i=1}^N \big[t_i = k\big] x_{i,j}}$.

~~~
We therefore form a Lagrangian (ignoring the first inequality constraint):
$$\begin{aligned}
ùìõ =& \textcolor{red}{‚àë_{i=1}^N ‚àë_{k=1}^K œÄ(‚Üíx_i)_k \log(œÄ(‚Üíx_i)_k)} \\
   & -\textcolor{magenta}{‚àë_{j=1}^D ‚àë_{k=1}^K Œª_{j,k} \Big(‚àë_{i=1}^N œÄ(‚Üíx_i)_k x_{i,j} - \big[t_i = k\big] x_{i,j}\Big)} \\
   & -\textcolor{darkgreen}{‚àë_{i=1}^N Œ≤_i \Big(‚àë_{k=1}^K œÄ(‚Üíx_i)_k - 1\Big)}.
\end{aligned}$$

# Derivation of Softmax using Maximum Entropy

We now compute partial derivatives of the Lagrangian, notably the values
$$\frac{‚àÇ}{‚àÇœÄ(‚Üíx_i)_k}ùìõ.$$

~~~
We arrive at
$$\frac{‚àÇ}{‚àÇœÄ(‚Üíx_i)_k}ùìõ = \textcolor{red}{\log(œÄ(‚Üíx_i)_k) + 1} - \textcolor{magenta}{‚Üíx_i^T ‚ÜíŒª_{*,k}} - \textcolor{darkgreen}{Œ≤_i}.$$

~~~
Setting the derivative of the Lagrangian to zero, we obtain
$$œÄ(‚Üíx_i)_k = \textcolor{red}{e}^{\textcolor{magenta}{‚Üíx_i^T ‚ÜíŒª_{*,k}} + \textcolor{darkgreen}{Œ≤_i} - \textcolor{red}{1}}.$$

~~~
Such a form guarantees $œÄ(‚Üíx_i)_k > 0$, which we did not include in the
conditions.

# Derivation of Softmax using Maximum Entropy

In order to find the $Œ≤_i$ values, we turn to the constraint
$$‚àë_k œÄ(‚Üíx_i)_k = ‚àë_k e^{‚Üíx_i^T ‚ÜíŒª_{*,k} +Œ≤_i-1} = 1,$$
~~~
from which we get
$$e^{Œ≤_i} = \frac{1}{‚àë_k e^{‚Üíx_i^T ‚ÜíŒª_{*,k} - 1}},$$

~~~
yielding
$$œÄ(‚Üíx_i)_k = e^{‚Üíx_i^T ‚ÜíŒª_{*,k} + Œ≤_i - 1} = \frac{e^{‚Üíx_i^T ‚ÜíŒª_{*,k}}}{‚àë_{k'} e^{‚Üíx_i^T ‚ÜíŒª_{*,k'}}} = \softmax(‚Üíx_i^T ‚áâŒª)_k.$$

# F-Score

---
# $F_1$-score

When evaluating binary classification, we have used **accuracy** so far.

~~~
![w=36%,f=right](true_false_positives.svgz)

However, there are other metrics we might want to consider. One of them is
$F_1$-score.

~~~
Consider the following **confusion matrix**:
| |Target positive|Target negative|
|-|---------------|---------------|
|Predicted<br>positive| **True Positive** (**TP**) | **False Positive** (**FP**) |
|Predicted<br>negative| **False Negative** (**FN**) | **True Negative** (**TN**) |

~~~
Accuracy can be computed as
$$\accuracy = \frac{\TP+\TN}{\TP+\TN+\FP+\FN}.$$

---
# $F_1$-score

![w=50%,h=center,mw=36%,f=right](true_false_positives.svgz)

| |Target positive|Target negative|
|-|---------------|---------------|
|Predicted<br>positive| **True Positive** (**TP**) | **False Positive** (**FP**) |
|Predicted<br>negative| **False Negative** (**FN**) | **True Negative** (**TN**) |

In some cases, we are mostly interested in positive examples.

~~~
![w=100%,h=center,mw=36%,f=right](precision_recall.svgz)

We define **precision** (the percentage of correct predictions in predicted examples)
and **recall** (the percentage of correct predictions in the gold examples) as
$$\begin{aligned}
  \precision =& \frac{\TP}{\TP+\FP}, \\
  \recall =& \frac{\TP}{\TP+\FN}.
\end{aligned}$$

---
# $F_1$-score

![w=46%,f=right](roc_pr.png)

Precision and recall go ‚Äúagainst each other‚Äù: decreasing the classifier
threshold usually increases recall and decreases precision, and vice versa.

~~~
We therefore define a single metric, the **$\boldsymbol{F_1}$-score**, as the harmonic mean of precision and recall:
$$\begin{aligned}
  F_1 =& \frac{2}{\precision^{-1} + \recall^{-1}} \\
      =& \frac{2 ‚ãÖ \precision ‚ãÖ \recall}{\precision + \recall} \\
      =&\frac{\TP~~~~~+~~~~~\TP}{\TP+\FP+\TP+\FN}.
\end{aligned}$$

---
# $F_1$-score and Other Means of Precision and Recall

![w=60%,f=right](fscore_mean_arithmetic.svgz)

Arithmetic mean of precision&recall is
$$\mathit{AM}(p, r) ‚âù \frac{p + r}{2}.$$

As any mean, it is ‚Äùbetween‚Äú the input values
$$\begin{gathered} \min(p, r) ‚â§ \mathit{AM}(p, r), \\ \mathit{AM}(p, r) ‚â§ \max(p, r).\end{gathered}$$

However,
$$\mathit{AM}(1\%, 100\%) = 50.5\%.$$

---
# $F_1$-score and Other Means of Precision and Recall

![w=60%,f=right](fscore_mean_geometric.svgz)

Geometric mean of precision&recall is
$$\mathit{GM}(p, r) ‚âù \sqrt{p ‚ãÖ r}.$$

It is better than the arithmetic mean, but still
$$\mathit{GM}(1\%, 100\%) = 10\%.$$

---
# $F_1$-score and Other Means of Precision and Recall

![w=60%,f=right](fscore_mean_harmonic.svgz)

Harmonic mean of precision&recall is
$$\mathit{HM}(p, r) ‚âù \frac{2}{\frac{1}{p} + \frac{1}{r}}.$$

In addition to being bounded by the input values, $\mathit{HM}$ is also
dominated by the minimum of its input values:
$$\begin{gathered} \min(p, r) ‚â§ \mathit{HM}(p, r), \\ \mathit{HM}(p, r) ‚â§ \max(p, r), \\ \mathit{HM}(p, r) ‚â§ 2\min(p, r).\end{gathered}$$

For example,
$$\mathit{HM}(1\%, 100\%) ‚âà 2\%.$$

---
# General $F_Œ≤$-score

The $F_1$-score can be generalized to the $F_Œ≤$-score, which can be used as a metric
when recall is $Œ≤$ times more important than precision; $F_2$ favoring recall
and $F_{0.5}$ favoring precision are commonly used.

~~~
The formula for $F_Œ≤$ is

$$\begin{aligned}
  F_Œ≤ =& \frac{1+Œ≤^2}{\precision^{-1} + Œ≤^2‚ãÖ\recall^{-1}} \\
      =& \frac{(1 + Œ≤^2) ‚ãÖ \precision ‚ãÖ \recall}{Œ≤^2 ‚ãÖ \precision + \recall} \\
      =& \frac{\TP~~~~~+~~~~~\hphantom{()}Œ≤^2‚ãÖ\TP}{\TP+\FP+Œ≤^2‚ãÖ(\TP+\FN)}.
\end{aligned}$$

# $F_1$-Score in Multiclass Classification

To extend $F_1$-score to multiclass classification, we expect one of the classes
to be _negative_ and the others to be _different kinds of positive_. For each of the
positive classes, we compute the same confusion matrix as in the binary case
(considering all other labels as negative ones), and then combine the results
in one of the following ways:

~~~
- **micro-averaged** $F_1$ (or just **micro** $F_1$): we first sum all the TP,
  FP and FN of the individual binary classifications and compute the final
  $F_1$-score (this way, the frequency of individual classes is taken into
  account);

~~~
- **macro-averaged** $F_1$ (or just **macro** $F_1$): we first compute
  $F_1$-scores of individual binary classifications and then compute
  an unweighted average (therefore, the frequency of classes is more
  or less ignored).
