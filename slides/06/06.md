~~~

# Lagrange Multipliers â€“ Inequality Constraints

Given a funtion $f(â†’x)$, we can find a maximum with respect to a vector
$â†’x âˆˆ â„^d$, by investigating the critical points $âˆ‡_â†’x f(â†’x) = 0$.

~~~

~~~

We start by again forming a Lagrangian $f(â†’x) + Î»g(â†’x)$.

~~~
The optimum can either be attained for $g(â†’x) > 0$, when the constraint is said

with the condition $Î»=0$.

~~~
When maximum is on a boundary, it corresponds to a critical point
with $Î»â‰ 0$ â€“ but note that this time the sign of the multiplier matters, because

Therefore, the solution to a maximization problem of $f(x)$ subject to $g(â†’x)â‰¥0$

$$\begin{aligned}

\end{aligned}$$

The above conditions are necessary conditions for a maximum (resp. minimum). However, it can be

- if the objective to optimize is a _concave_ function (resp. _convex_ for minimization);

$$\argmin_{w,b} \frac{1}{2} ||â†’w||^2 \textrm{~given that~~}t_i y(â†’x_i) â‰¥ 1,$$

# Support Vector Machines for Non-linearly Separable Data

![w=28%,f=right](svm_softmargin.svgz)

Until now, we assumed the data to be linearly separable â€“ the  
**hard-margin SVM** variant. We now relax this condition to arrive at
**soft-margin SVM**.
~~~
The idea is to allow points to be in the margin or even on the _wrong side_ of
the decision boundary. We introduce _slack variables_ $Î¾_i â‰¥ 0$, one for each
training instance, defined as
$$Î¾_i = \begin{cases} 0 &\textrm{~for points fulfilling~}t_i y(â†’x_i) â‰¥ 1,\\
                      |t_i - y(â†’x_i)|&\textrm{~otherwise}.\end{cases}$$

~~~
Therefore, $Î¾_i=0$ signifies a point outside of margin, $0 < Î¾_i < 1$ denotes
a point inside the margin, $Î¾_i=1$ is a point on the decision boundary and
$Î¾_i>1$ indicates the point is on the opposite side of the separating
hyperplane.

Therefore, we want to optimize
$$\argmin_{w,b} C âˆ‘_i Î¾_i + \frac{1}{2} ||â†’w||^2 \textrm{~given that~~}t_i y(â†’x_i) â‰¥ 1 - Î¾_i\textrm{~and~}Î¾_i â‰¥ 0.$$

At its core, the SMO algorithm is just a coordinate descent.

~~~

# Sequential Minimal Optimization Algorithm

The SMO is an efficient algorithm, because we can compute the update to
$a_i, a_j$ efficiently, because there exists an closed form solution.

~~~
Assume that we are updating $a_i$ and $a_j$. Then using the condition $âˆ‘_k a_k t_k = 0$ we can
write $a_i t_i = -âˆ‘_{kâ‰ i} a_k t_k$. Given that $t_i^2 = 1$ and denoting $Î¶=-âˆ‘_{kâ‰ i, kâ‰ j} a_k t_k$, we get
$$a_i = t_i (Î¶-a_j t_j).$$

~~~
Minimizing $L(â†’a)$ with respect to $a_i$ and $a_j$ then amounts to minimizing
a quadratic function of $a_j$, which has an analytical solution.

~~~
Note that the real SMO algorithm has several heuristics for choosing $a_i, a_j$
such that the $L$ can be minimized the most.

# Sequential Minimal Optimization Update Rules

We already know that $a_i = t_i (Î¶-a_j t_j).$

~~~
To find $a_j$ optimizing the loss $L$, we use the formula for locating a vertex
of a parabola
$$a_j^\textrm{new} â† a_j - \frac{âˆ‚L/âˆ‚a_j}{âˆ‚^2L/âˆ‚a_j^2},$$
which is in fact one Newton-Raphson iteration step.

~~~
Denoting $E_j â‰ y(â†’x_j) - t_j$, we can compute the first derivative as
$$\frac{âˆ‚L}{âˆ‚a_j} = t_j (E_i - E_j)$$
and the second derivative as
$$\frac{âˆ‚^2L}{âˆ‚a_j^2} = 2K(â†’x_i, â†’x_j) - K(â†’x_i, â†’x_i) - K(â†’x_j, â†’x_j).$$

---

If the second derivative is positive, we know that the vertex is really

$$a_j^\textrm{new} â† a_j - t_j\frac{E_i - E_j}{2K(â†’x_i, â†’x_j) - K(â†’x_i, â†’x_i) - K(â†’x_j, â†’x_j)}.$$

$$\begin{aligned}

\end{aligned}$$

~~~
Finally, we set
$$a_i^\textrm{new} â† a_i - t_i t_j\big(a_j^\textrm{new} - a_j).$$

To arrive at the bias update, we consider the KKT condition that
for $0 < a_j^\textrm{new} < C$ it must hold that $t_j y(â†’x_j) = 1$. Combining it with
with $b = E_j + t_j - âˆ‘_l a_l t_l K(â†’x_j, â†’x_l)$, we get the following
value
$$b_j = b - E_j - t_i (a_i^\textrm{new} - a_i)K(â†’x_i, â†’x_j) - t_j (a_j^\textrm{new} - a_j)K(â†’x_j, â†’x_j).$$

~~~
Analogously for $0 < a_i^\textrm{new} < C$ we get
$$b_i = b - E_i - t_i (a_i^\textrm{new} - a_i)K(â†’x_i, â†’x_i) - t_j (a_j^\textrm{new} - a_j)K(â†’x_j, â†’x_i).$$

~~~
Finally, if $a_j^\textrm{new}, a_i^\textrm{new} âˆˆ \{0, C\}$, we know that all values between $b_i$ and $b_j$ fulfil the KKT conditions.
We therefore arrive at the following update for bias:
$$b^\textrm{new} = \begin{cases}
  b_i & \textrm{if~~} 0 < a_i^\textrm{new} < C \\
  b_j & \textrm{if~~} 0 < a_j^\textrm{new} < C \\
  \frac{b_i + b_j}{2} & \textrm{otherwise}.
\end{cases}$$
